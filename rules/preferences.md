# Preferences — структура и визуал

## Разбивка файлов (веб-проекты)

Сайты/веб-страницы — ВСЕГДА разбивать на отдельные файлы:
```
project/
├── index.html       ← только разметка, никакого <style> и <script>
├── css/
│   └── style.css    ← все стили
└── js/
    └── main.js      ← весь JavaScript
```

**Зачем:** при правке CSS читается только style.css (~400 строк) вместо монолита (~2000 строк) → экономия токенов в 5× при итерациях.

**Правило для себя:**
- Создаю монолит? → спросить "правок будет много?" → если да, разбивать сразу
- `<style>` внутри html → только для единственного уникального правила, которое не повторится (например, кастомный inline цвет)
- `<script>` внутри html → только для инлайн данных (JSON-LD, конфигурация)

## CSS dropdown — правило

Dropdown через `:hover` с зазором `top: calc(100% + Npx)` → **hover рвётся** при переходе мышки через зазор.

**Правильный паттерн:**
```css
.dropdown {
  top: 100%;                   /* вплотную */
  padding-top: 8px;            /* визуальный зазор через padding */
  /* hover не теряется т.к. dropdown технически под ссылкой */
}
```
Не использовать `top: calc(100% + Npx)` — заменять на `padding-top`.

## Цвет / визуал

- Насыщенные, живые цвета — не зажимать без причины

- Псевдографика везде как дефолт (таблицы, схемы, дерево)

## BAT vs Python — глобальное правило

**Python предпочтительнее BAT везде где есть логика.**

```
BAT — только один файл menu.bat на проект:
  └── меню с цифрами → запускает python-скрипты

Всё остальное — Python:
  setup.py     ← установка зависимостей
  start.py     ← запуск приложения
  stop.py      ← остановка
  start_debug.py ← режим отладки
```

**Почему:**
- BAT ломается на скобках в echo: `(текст)` → краш
- BAT ломается на кириллице без `chcp 65001`
- Python кроссплатформенный, читаемый, отлаживаемый
- Python ловит ошибки и выводит понятно

**Правило при создании нового проекта:**
- Нужен скрипт запуска? → `start.py`
- Нужна установка? → `setup.py`
- Один `menu.bat` как единственная точка входа — ок
- Больше одного bat файла → вопрос "зачем не Python?"

## Структура файлов проекта

```
src/          ← исходники (свободно трогать)
docs/         ← документация (дополнять, не переписывать)
assets/       ← ресурсы (не трогать без явного запроса)
logo/         ← логотип (не трогать)
```
