## Принцип

Многошаговые и архитектурные задачи требуют Plan Mode.
Хук автоматически определяет сложность промпта и инжектирует требование.

## Триггеры Plan Mode (автоматически)

```
· "создай проект / систему / приложение / бот / сайт"
· "разработай" / "реализуй" / "с нуля" / "от нуля"
· "рефакторинг" / "архитектура" / "миграция"
· 3+ императивных глагола в одном промпте
· "также" / "кроме того" + 2+ действия
· 4+ шагов в нумерованном плане → Stop хук
```

## Что делать при [PLAN_MODE_REQUIRED]

1. Вызвать `EnterPlanMode`
2. Написать план в файл (указывается автоматически)
3. Дождаться одобрения через `ExitPlanMode`
4. Только потом выполнять

## Что НЕ триггерит Plan Mode

```
· Вопросы ("как работает X?", "объясни Y")
· Read-only задачи (найди, покажи, перечисли)
· Точечные фиксы ("исправь только эту строку")
· Короткий промпт < 40 символов
```
