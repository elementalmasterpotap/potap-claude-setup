> Говоришь что хочешь увидеть — AI делает. Ты деплоишь и смотришь.

Работает для любого проекта где есть настраиваемый результат: шейдер, UI, текст, логика.
Адаптируй маппинг под свой домен (см. раздел "Как адаптировать").

---

## Принцип

```
промпт на человеческом языке
    │
    ▼
AI находит нужные константы / параметры / код
    │
    ▼
деплой одной командой
    │
    ▼
смотришь/проверяешь результат
    │
    ├─ нравится ──────────────── готово
    │
    └─ не нравится ─── новый промпт или откат
```

---

## Хорошие промпты (примеры)

```
"сделай X теплее / холоднее / ярче / тише"
"хочу чтобы Y было плавнее"
"слишком агрессивно, приглуши"
"добавь больше Z"
"убери [эффект] совсем"
"хочу больше контраста между A и B"
"[эффект] слишком заметный, сделай тоньше"
```

## Что AI НЕ трогает (заморозить в проектном CLAUDE.md)

- Критичные зависимости которые ломались при изменении
- Пороги и топология детекторов / автоматических систем
- Проверенные балансные значения которые нашли итерацией

---

## Откат

Всегда иметь два уровня:
```
# К последнему рабочему состоянию:
[команда восстановления prev]

# К эталону:
[команда восстановления backup]
```

`prev` — обновляется автоматически при каждом деплое.
`backup` — трогается только явной командой.

---

## Как адаптировать под новый проект

1. Заполни таблицу маппинга: "хочу X" → "AI меняет параметр Y"
2. Опиши команду деплоя в разделе "цикл"
3. Зафиксируй что НЕЛЬЗЯ трогать
4. Добавь способ проверки результата (что именно смотреть)
