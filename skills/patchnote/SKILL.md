Напиши патчнот и вставь его в начало `docs/PATCHNOTES.md`.

Версия и изменения: $ARGUMENTS
(если не указано — определи по последним изменениям в коде / git diff)

## Формат (СТРОГО):

```
## Патч vX.X.X — YYYY-MM-DD
*Tagline — одна строка суть*

[Симптом / контекст — что было не так или зачем патч]

[Что сделал — что пользователь теперь увидит]

[Список с тире — только если несколько отдельных изменений]
```

## Правила тона:
- От первого лица: "убрал", "прикрутил", "поймал", "добавил"
- Tagline ОБЯЗАТЕЛЕН — курсив, одна строка
- Без техжаргона — человеческим языком
- Финальная фраза-вывод желательна
- Мат ок, неформально

## После написания:
Вставь патчнот в начало `docs/PATCHNOTES.md` (сразу после заголовка `# Патчноты` если он есть).
