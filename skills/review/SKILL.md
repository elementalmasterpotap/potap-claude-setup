# /review — Система самообучения

**Фокус:** $ARGUMENTS (если пусто — анализируй все категории)

## Алгоритм

### 1. СБОР ДАННЫХ

Прочитай (параллельно):
- `~/.claude/tasks/SESSION_PATTERNS.md` — пассивный лог паттернов из session-pattern-logger.py
- `~/.claude/SESSION.md` — лог файлов из checkpoint.py

Проанализируй текущий диалог в памяти — ищи:
- Что пользователь исправлял после твоего ответа ("нет", "не так", "неправильно")
- Что Claude делал несколько раз одинаково (повторяющийся подход)
- Какие команды/паттерны запускались 2+ раз
- Что пользователь говорил о своих предпочтениях
- Где не хватало инструмента (приходилось делать руками или через костыль)

**Отдельно — поиск потерь токенов** (чеклист, проверять всегда):

```
ФАЙЛЫ:
  □ Читался файл >200 строк целиком, а правка была точечной?
    → Read с offset/limit дал бы 10-20× экономию
  □ Один файл читался 2+ раз за сессию?
    → Структура проекта не изолирует изменяемые части
  □ Файл выпал из контекста после compaction ("too large to include")?
    → Монолит превысил порог — нужна разбивка на модули
  □ Читался весь bot.py / main.py ради одной функции?
    → Нужна структура handlers/ utils/ persistence/

ВЫВОД:
  □ Bash команда вернула >50 строк в main context?
    → Должен быть субагент (verbose ops → subagent)
  □ pip install / npm install вывод попал в контекст?
    → setup.py с capture_output=True или субагент

ИНСТРУМЕНТЫ:
  □ Использовался Sonnet для "найди файл" / "прочитай и суммируй"?
    → Haiku субагент дал бы -70% токенов
  □ MCP серверы были загружены но не использовались?
    → Отключить через /mcp до следующей сессии

СТРУКТУРА ПРОЕКТА:
  □ Есть монолитный файл >300 строк который правился 2+ раз?
    → Предложить разбивку: каждый модуль < 150 строк
  □ CSS/JS внутри HTML, или SQL внутри Python, или GUI внутри логики?
    → Изоляция по типу = читаем только нужный файл
  □ Нет .claudeignore при наличии node_modules / *.lock / dist?
    → +30-100K токенов в холостую при каждом glob/grep
```

**Отдельно — Haiku-специфический чеклист** (проверять всегда):

```
МАРШРУТИЗАЦИЯ:
  □ Задача была read-only (поиск/анализ/структура) но Haiku НЕ предложен?
    → Добавить тип задачи в haiku-economy.md таблицу + haiku-suggest.py сигналы
  □ Задача содержала "найди и исправь" — Sonnet использовался корректно?
    → Если нет — напомнить: смешанные задачи всегда Sonnet
  □ Verbose output от анализа попал в main context (не в субагент)?
    → Паттерн субагента: summary в main, verbose там

СТРУКТУРА ПОД HAIKU:
  □ Создан новый файл >150 строк без плана разбивки?
    → Сейчас монолит, через 2-3 правки выпадет из контекста
    → Предложить: для этого типа проекта — шаблон из haiku-economy.md
  □ Проект без .claudeignore и есть node_modules / __pycache__ / build/?
    → Создать .claudeignore (шаблон в haiku-economy.md)
  □ Главный файл проекта >200 строк и правится регулярно?
    → Кандидат на разбивку — описать конкретную структуру (не абстрактно)

СЕТАП CLAUDE (~/. claude/):
  □ Какой-то rules/*.md вырос >200 строк?
    → Разбить: детали в rules/sub/, суть в rules/main.md
  □ MEMORY.md близко к 200 строкам (обрезается)?
    → Compact старые записи, вынести в отдельный файл
  □ Новый rules/*.md добавлен без Stop хука?
    → Проверить нужен ли автоматический enforcer

ПЕРЕХОД ПРОЕКТА НА HAIKU:
  □ Есть монолит >300 строк → план разбивки:
      Шаг 1: определить модули (config / logic / ui / utils)
      Шаг 2: создать пустые файлы с правильными именами
      Шаг 3: перенести код по одному модулю
      Шаг 4: обновить импорты
      Шаг 5: убедиться каждый файл < 150 строк
  □ .claudeignore создан для проекта? (шаблон: haiku-economy.md)
```

### 2. ФОРМИРОВАНИЕ ПРЕДЛОЖЕНИЙ

Для каждого найденного паттерна сформируй запись:

```
тип        — [ХУК] / [ПРАВИЛО] / [СКИЛЛ] / [ПАМЯТЬ] / [ПЛАГИН]
что        — конкретное предложение (1 строка)
пример     — что именно произошло в сессии (цитата или описание)
рекомендация — ГЛОБАЛЬНО или ЛОКАЛЬНО + почему
```

**Правила категоризации:**

| Если... | То тип | Куда |
|---------|--------|------|
| Claude делал X неправильно 2+ раз | ХУК (Stop) | глобально |
| Обсудили новое соглашение/правило | ПРАВИЛО | зависит от специфики |
| Команда/паттерн повторялась 3+ раз | СКИЛЛ | глобально |
| Потап выразил предпочтение | ПАМЯТЬ | MEMORY.md |
| Приходилось делать вручную то что мог инструмент | ПЛАГИН | глобально |
| Новая конвенция специфична для проекта | ПРАВИЛО | локально |
| Read-only задача решалась Sonnet без Haiku | HAIKU | haiku-economy.md + haiku-suggest.py |
| Новый тип read-only задачи найден в сессии | HAIKU | добавить сигнал в haiku-suggest.py |
| Проект без .claudeignore | HAIKU | создать .claudeignore (шаблон в haiku-economy.md) |
| Монолитный файл >150 строк создан/правился 2+ раз | HAIKU | план разбивки по шаблону типа проекта |
| Проект нового типа → нет шаблона модулей | HAIKU | добавить шаблон в haiku-economy.md |
| Найдена потеря токенов из чеклиста (1+ пункт) | ТОКЕНЫ | глобально или локально |

**ГЛОБАЛЬНО** = `~/.claude/` — работает во всех проектах
**ЛОКАЛЬНО** = `.claude/` в текущем проекте

### 3. ВЫВОД ТАБЛИЦЫ

Покажи таблицу предложений (все найденные, приоритет сверху):

```
╔══════════════════════════════════════════════════════════════════════╗
║  /review — результаты анализа сессии                                ║
╠═══╤══════════╤══════════════════════════════════════╤═══════════════╣
║ # │ Тип      │ Предложение                          │ Куда          ║
╠═══╪══════════╪══════════════════════════════════════╪═══════════════╣
║ 1 │ ХУК      │ Блокировать X если Y                │ глобально     ║
║ 2 │ ПРАВИЛО  │ Добавить в communication.md: Z       │ глобально     ║
║ 3 │ СКИЛЛ    │ Создать /deploy-report               │ глобально     ║
║ 4 │ ПАМЯТЬ   │ Потап предпочитает A в ситуации B    │ MEMORY.md     ║
╚═══╧══════════╧══════════════════════════════════════╧═══════════════╝
```

Под таблицей — краткий комментарий по каждому пункту (пример из сессии).

Если предложений 0 — сказать честно: "Чистая сессия, паттернов для фиксации нет."

### 4. ИНТЕРАКТИВНЫЙ ВЫБОР

Используй AskUserQuestion с multiSelect: true.

Если предложений ≤ 4:
- Один вопрос: все предложения как варианты + "Пропустить всё"

Если предложений > 4:
- Первый раунд: топ-4 по приоритету (хуки > правила > скиллы > память)
- После применения спросить: "Показать ещё N предложений?"

**Формулировка вопроса:** "Что применяем из этой сессии?"

Для каждого выбранного предложения с неочевидным местом (ГЛОБАЛЬНО vs ЛОКАЛЬНО):
- Задать отдельный вопрос или включить в формулировку варианта

### 5. ПРИМЕНЕНИЕ

Выполняй последовательно для каждого выбранного:

**[ХУК]:**
- Если паттерн простой (regex в тексте ответа) → создай `~/.claude/scripts/<name>-check.py`
  по образцу `anti-ration.py` (stdin JSON → проверка → block/warn)
- Добавь в settings.json в массив Stop[]
- Или: вызови `/hookify <name>` если достаточно файла-правила

**[ПРАВИЛО]:**
- Определи по карте правил из CLAUDE.md куда писать
- Edit нужного файла — добавить сжато, без воды
- Если новый файл → создать `rules/<name>.md`

**[СКИЛЛ]:**
- Создать `~/.claude/skills/<name>/SKILL.md` (или `.claude/skills/<name>/SKILL.md` если локально)
- Тип: `disable-model-invocation: true` (action) или `user-invocable: false` (knowledge)

**[ПАМЯТЬ]:**
- Edit `~/.claude/memory/MEMORY.md`
- Сначала проверить нет ли уже похожей записи (избегать дублей)

**[ПЛАГИН]:**
- Объяснить как добавить в settings.json mcpServers
- Указать конкретное имя пакета если известно
- НЕ добавлять автоматически — только рекомендовать

**[ТОКЕНЫ]:**

| Источник потери | Действие |
|---|---|
| Монолит >300 строк читался 2+ раз | Предложить структуру модулей, описать разбивку в haiku-economy.md |
| Read без offset/limit на большом файле | Напомнить паттерн Grep→offset→limit, добавить в haiku-economy.md если нет |
| Bash вывод >50 строк в main context | Создать субагент-паттерн для этой задачи |
| Sonnet для поиска/анализа | Добавить в Haiku-eligible список (haiku-economy.md) |
| Нет .claudeignore при node_modules | Создать `.claudeignore` в проекте |
| MCP сервер без использования | Рекомендовать `/mcp` отключение |
| Файл выпал из контекста после compaction | ЛОКАЛЬНО: предложить разбивку + добавить в .claude/CLAUDE.md примечание о структуре |

Для структурных проблем (монолит): описать конкретную разбивку с именами файлов — не абстрактно.

### 6. ФИНАЛ

После применения всего выбранного:

1. Если изменились `CLAUDE.md`, `settings.json`, `rules/*.md` → запустить:
   ```
   python3 ~/.claude/update_telegraph.py
   ```

2. Показать итог:
   ```
   ╔══════════════════════════════════════╗
   ║  Применено: N из M предложений      ║
   ╠══════════════════════════════════════╣
   ║  ✓ [ХУК] название                  ║
   ║  ✓ [ПРАВИЛО] название              ║
   ║  ✗ [СКИЛЛ] пропущено              ║
   ╚══════════════════════════════════════╝
   ```

3. Очистить SESSION_PATTERNS.md (применённые паттерны уже зафиксированы):
   ```
   # Оставить заголовок, удалить строки которые были применены
   ```

## Приоритет предложений

```
1. ХУК        — если Claude делал что-то неправильно → блокировать сразу
2. ПРАВИЛО     — соглашения которые важны для будущей работы
3. ТОКЕНЫ     — потери найдены чеклистом → структурные улучшения проекта
4. СКИЛЛ      — экономия времени на повторяющихся паттернах
5. ПАМЯТЬ     — факты о Потапе которые нужны в следующих сессиях
6. ПЛАГИН     — рекомендация, не автоматическое действие
```

## Что НЕ предлагать

- Не дублировать то что уже есть в rules/*.md / CLAUDE.md / MEMORY.md
- Не предлагать хук если паттерн был один раз (нужно 2+)
- Не предлагать скилл для одноразовой команды
- Не предлагать плагин без чёткого use-case из сессии
