# /compact-smart — умное сжатие сессии

Pro тариф: ~44K токенов / 5 часов. Каждый длинный обмен сокращает ресурс.
`/compact` без аргументов теряет детали. `/compact-smart` сохраняет что нужно.

## Что делать при вызове /compact-smart

1. **Собери summary** перед compact — выведи в ответ:

```
## Что сохраняем (перед compact)

Задача:        [одна строка — что делали]
Изменены:      [список файлов: путь + что изменено]
Незавершено:   [если есть]
Ключевые реш.: [архитектурные договорённости]
Следующий шаг: [что делать после compact]
```

2. **Запусти compact** с кастомными инструкциями:

```
/compact Сохрани: (1) список всех изменённых файлов с описанием изменений,
(2) незавершённые задачи, (3) ключевые архитектурные решения из диалога,
(4) следующий шаг. Остальное сожми максимально.
```

3. После compact — **восстанови контекст** из summary выше.

## Когда вызывать

```
Сигналы что пора compact:
  · Сессия идёт > 1 часа
  · Ответы становятся длиннее (контекст пухнет)
  · Переходишь к новой задаче в том же проекте
  · "context left until auto-compact" < 40%
  · Ты видишь полосу "Compacting..." — compact уже сработал сам

Когда /clear лучше:
  · Задача полностью другая (другой проект, другая фича)
  · Ничего из текущего диалога не нужно дальше
  · Хочешь начать с нуля
```

## Тайминг Pro окон

```
Pro: ~44K токенов / окно · ~45 сообщений / окно · сброс каждые 5 часов
Окно = rolling от ПЕРВОГО сообщения (не от полуночи)

Стратегия:
  07:00  первое сообщение (даже "hi") → запускает окно
  07:00–12:00  первое рабочее окно
  12:00–17:00  второе окно
  17:00–22:00  третье окно
  = 3 полных рабочих окна за день
```

## Модели и стоимость

```
Sonnet  → default для 80%+ задач
Opus    → только: сложная архитектура, multi-file рефакторинг
Haiku   → read-only анализ [→ Haiku], быстрые вопросы
Thinking → только когда реально нужно глубокое рассуждение
         → для "найди файл" / "покажи структуру" — ВЫКЛ
```
